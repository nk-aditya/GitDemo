## Forms in HTML
Forms are used to collect user input. They can contain various types of input elements, such as text fields, checkboxes, radio buttons, and submit buttons.

### Basic Form Structure
A basic form is created using the <form> tag.
- action : attribute specifies where to send the form data when the form is submitted.
- method : attribute specifies the HTTP method to use when sending form data (GET or POST).
-- GET > sends data in the URL (useful for simple searches)
-- POST > sends data in the request body (useful for sensitive data)
- placeholder : attribute provides a hint to the user about what to enter in the input field. It is displayed inside the input field when it is empty and disappears when the user starts typing.

### Type of Input
  • text: A single-line text input field.
  • password: A single-line text input field that hides the entered characters.
  • email: An input field for email addresses with built-in validation.
  • number: An input field for numeric values with optional min and max attributes.
  • date: An input field for selecting a date.
  • checkbox: A checkbox that can be toggled on or off.
  • radio: A radio button that allows users to select one option from a group.
  • color: An input field for selecting a color.
  • file: An input field that allows users to upload files.
  • submit: A button that submits the form data to the server.
  • reset: A button that resets all form fields to their default values.
  • hidden: An input field that is not visible to the user but can store data to be submitted with the form.

### Form Validation
HTML5 provides built-in form validation features. You can use attributes like required, minlength, maxlength, pattern, and more to specify validation rules for form fields.
- required : makes the field mandatory.
- minlength : specifies the minimum number of characters allowed.
- maxlength : specifies the maximum number of characters allowed.
